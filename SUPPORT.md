# Support

## Before opening an issue

- Read `README.md` for API contracts, delivery guarantees, and deployment boundaries.
- Run the commands in `CONTRIBUTING.md` against Go `1.26.5`.
- Search existing issues for the same symptom or proposal.

## Where to ask

- Use the bug report form for reproducible defects.
- Use the feature request form for public API or behavior proposals.
- Use GitHub security advisories for vulnerabilities. Do not disclose security issues in a
  public issue.

Include a minimal reproduction, dependency versions, sanitized logs, and the deployment
shape. For multi-instance behavior, state replica count, discovery backend, coordinator,
and Redis or Valkey version when used.
